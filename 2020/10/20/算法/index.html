算法

Author: 舒克 Date: October 20, 2020  15:24:20 Category: Programme

==version 1.1==

绪论

程序设计的实质：数据表示（存储在内存中，核心：数据结构）和数据处理（核心：算法）

数值问题 -> 数学问题

非数值问题 -> 数据结构

数据结构

数据：信息的载体

数据元素：数据的基本单位（讨论数据结构时涉及的最小数据单位）

数据项：构成数据元素的不可分割的最小单位

数据结构：相互之间存在一定关系的数据元素的集合

数据的逻辑结构

面向问题

是数据元素之间逻辑关系的整体

用来反映数据元素之间的关联方式或邻接关系

抽象的数据模型 逻辑关系
线性表 一对一
树 一对多
图 多对多
集合 除了同属一个集合外没有任何关系

数据的存储结构

面向计算机

用来存储数据元素并显示或隐式地存储数据元素之间的逻辑关系

顺序存储：由元素的存储位置表示逻辑关系

链式存储：用指针表示逻辑关系

算法

是对特定问题求解步骤的一种描述，是指令的有限序列

特性

输入：可以没有

输出：必须有

有穷性：有穷步在有穷时间内完成

确定性：每条指令有确切含义

可行性：通过已经实现的基本操作执行有限次来实现

好算法

正确性：合法输入得到正确结果

鲁棒性：对非法输入的抵抗能力

简单性：容易理解和实现

抽象分级：每个逻辑步骤能模块调用

高效性：时、空效率高

描述方法

语言 优点 缺点
自然语言 容易理解 有二义性
流程图 直观易懂 严密性不如程序语言，灵活性不如自然语言
程序设计语言 能直接执行 抽象性差，拘于细节
伪代码 算法语言 真没缺点

算法分析

事后统计

事前分析估算：渐进复杂度

问题规模

输入量的多少（n）

Eg：找到100以内的素数，问题规模为100

基本语句：执行次数与整个算法的执行次数成正比的语句

渐进时间复杂度 O(n)

平均情况：假设数据等概率分布

O(log2 n)<O(n)<O(n*log2 n)<O(n^2)

Eg：三层循环 O(n^3)

线性表 L=(a1, …, an)

长度：线性表的长度（有限序列）

空表 L=( )：长度为0

基本操作

InitList

DestroyList

Length

Get 按位查找

Locate 按值查找

Insert

Delete

Empty

特性

有限性：数据元素个数有穷

向同性：元素类型统一

顺序性：存在序偶关系，每个元素有且仅有一个前驱后继

顺序表

用地址连续的存储单元依次存储线性表的数据元素，通常用一维数组实现

静态存储分布

基地址：起始地址

数组长度 >= 线性表长度

存取结构：随机存取

优缺点

优点

无需为表示元素间的逻辑关系新增空间

随机存取可以快速存取任意位置元素

缺点

插入删除需要移动大量元素

表容量难确定，难扩充

造成存储空间碎片

单链表

单链表只有一个指针域

存储特点：

逻辑次序和物理次序不一定相同

元素之间的逻辑关系用指针表示

存取结构：顺序存取

结点

数据元素的存储映像

头结点：第一个元素结点之前附设的结点，以便空表和非空表处理统一

首结点：

指针

存储的后继元素的地址信息

头指针：指向第一个元素所在的结点，起标识一个单链表的作用

尾标志：最后一个元素所在的终端结点的指针域为NULL

其他链表

循环链表

附设头结点，空表和非空表处理一致

没有明显尾端（P != first）

开始结点：first->next

带尾指针时

开始结点：rear->next->next

终端结点：rear

双链表

快速求直接前驱

空间换时间

静态链表

间接寻址：结合数组和指针，将数组中存储数据元素的单元改为存储指向该元素的指针

优点：插入删除时只需要修改指针不移动元素

缺点：没解决连续存储分配的问题，还需要维护一个空闲链，不能随机存取

对比

时间性能

实现基于某存储结构的基本操作的时间复杂度

空间性能

存储密度 = 数据域占用的存储量 / 整个结点占用的

顺序表 = 1 存储空间利用率较高

链表 < 1

栈和队列

操作受限的线性表，逻辑结构相同

栈

限定仅在表尾插入删除的线性表，单一的前驱和后继

特性：后进先出

栈顶：允许插入和删除

栈空：不含任何数据元素的栈

ADT

Push

Pop

GetTop

Empty

顺序栈

栈空：top = -1

栈满：top = StackSize - 1

两栈共享空间

栈空：top1 = - 1 top2 = Stack_Size

栈满：top2 = top1 + 1 top1 = top2 - 1

链栈

以单链表的头部做栈顶，不需要附设头结点

头插入栈

顺序栈vs链栈

基本操作的算法只需要常数时间

链栈有结构性开销，应采用顺序栈

队列

只允许在一端插入另一端删除的线性表

特性：先进先出

队尾：允许出队（删除）

设置队头、队尾两个指针，改进出队时间性能

队头指针：头元素前一个位置

队尾指针：队尾元素

ADT

EnQueue

DeQueue

GetQueue

Empty

顺序存储-循环队列

假溢出：元素插入数组下标较大的位置移过去，数组的低端还有空闲空间

队空：front = rear = QueueSize - 1

队满：( rear + 1 ) % QueueSize = front

（头尾）进一：rear = ( rear + 1 ) % QueueSize

链队列

使用头结点，使操作统一

字符串和多维数组 S=“s1 s2 … sn”

是零个或多个字符组成的有限序列，是数据元素为单个字符的线性表
空格串：包含空格的串
子串：任意个连续的字符组成的子序列
主串：包含子串的串
子串在主串中的位置：子串的第一个字符在主串中的序号
字符串的比较：由ASCII码决定

模式匹配

在主串S中寻找子串T的过程
模式：子串T

BF算法

最好情况下时间复杂度O(n+m)
改进的KMP算法：
i不回溯，j回溯到next[j]
失败时 T[0]T[k-1]=S[i-k]S[i-1]
成功时 T[j-k]T[j-1]=S[i-k]S[i-1]
综上 T[0]T[k-1]=T[j-k]T[j-1]

多维数组

类型相同的数据元素构成的有序集合
行[l1,h1]、列[l2,h2]：loc=基地址+c((i-l1)(h2-l2+1)+(j-l2))

矩阵的压缩存储

行优先：以行序为主序
特殊矩阵：很多值相同的元素且他们的分布有规律，稀疏矩阵不属于特殊矩阵
压缩存储：值相同的元素只分配一个存储空间，零元素不分配存储空间

对称矩阵

关于主对角线对称，只存储下三角部分（和主对角线）
n*n -> n(n+1)/2
存在数组SA[n(n+1)/2]中，下标k=i(i-1)/2+j-1（下标从0开始）

三角矩阵

对称矩阵，对角线上为常数c
n*n -> n(n+1)/2+1
存在数组SA[n(n+1)/2+1]中，下标为k（从0开始）

下三角

i >= j k = i(i-1)/2+j-1
i < j k = n(n+1)/2

上三角

i <= j k = (i-1)(2n-i+2)/2+j-i
i > j k = n(n+1)/2

对角矩阵

k = 2 + 3(i-2) + (j-i+2)

稀疏矩阵

三元组顺序表
row，col，item
在顺序表倒数三行需要存矩阵的行、列数、非零元的个数
十字链表
row，col，item，right，down
right 同行下一个三元组
down 同列下一个三元组

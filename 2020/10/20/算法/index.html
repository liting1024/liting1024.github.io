绪论

程序设计的实质：数据表示（存储在内存中，核心：数据结构）和数据处理（核心：算法）

数值问题 -> 数学问题

非数值问题 -> 数据结构

数据结构

数据：信息的载体

数据元素：数据的基本单位（讨论数据结构时涉及的最小数据单位）

数据项：构成数据元素的不可分割的最小单位

数据结构：相互之间存在一定关系的数据元素的集合

数据的逻辑结构

面向问题

是数据元素之间逻辑关系的整体

用来反映数据元素之间的关联方式或邻接关系

抽象的数据模型 逻辑关系
线性表 一对一
树 一对多
图 多对多
集合 除了同属一个集合外没有任何关系

数据的存储结构

面向计算机

用来存储数据元素并显示或隐式地存储数据元素之间的逻辑关系

顺序存储：由元素的存储位置表示逻辑关系

链式存储：用指针表示逻辑关系

算法

是对特定问题求解步骤的一种描述，是指令的有限序列

特性

输入：可以没有

输出：必须有

有穷性：有穷步在有穷时间内完成

确定性：每条指令有确切含义

可行性：通过已经实现的基本操作执行有限次来实现

好算法

正确性：合法输入得到正确结果

鲁棒性：对非法输入的抵抗能力

简单性：容易理解和实现

抽象分级：每个逻辑步骤能模块调用

高效性：时、空效率高

描述方法

语言 优点 缺点
自然语言 容易理解 有二义性
流程图 直观易懂 严密性不如程序语言，灵活性不如自然语言
程序设计语言 能直接执行 抽象性差，拘于细节
伪代码 算法语言 真没缺点

算法分析

事后统计

事前分析估算：渐进复杂度

问题规模

输入量的多少（n）

Eg：找到100以内的素数，问题规模为100

基本语句：执行次数与整个算法的执行次数成正比的语句

渐进时间复杂度 O(n) P15

平均情况：假设数据等概率分布

O(log[2] n)<O(n)<O(n*log[2] n)<O(n^2)

Eg：

1、三层循环 O(n^3)

2、for(int i=1;i<=n;i=2*i)

​ x++;

2^(T(n))<=n

T(n)<=log[2] n

3、for(int i=1;i<=n;i++)

​ for(int j=1;j<=i-1;j++)

​ x++;

求和（i-1）= n(n-1)/2

线性表 L=(a1, …, an)

长度：线性表的长度（有限序列）

空表 L=( )：长度为0

特性

有限性：数据元素个数有穷

向同性：元素类型统一

顺序性：存在序偶关系，每个元素有且仅有一个前驱后继

顺序表

用地址连续的存储单元依次存储线性表的数据元素，通常用一维数组实现

静态存储分布

基地址：起始地址

数组长度 >= 线性表长度

存取结构：随机存取

优缺点

优点

无需为表示元素间的逻辑关系新增空间

随机存取可以快速存取任意位置元素

缺点

插入删除需要移动大量元素

表容量难确定，难扩充

造成存储空间碎片

单链表

单链表只有一个指针域

存储特点：

逻辑次序和物理次序不一定相同

元素之间的逻辑关系用指针表示

存取结构：顺序存取

结点

数据元素的存储映像

头结点：第一个元素结点之前附设的结点，以便空表和非空表处理统一

存储的后继元素的地址信息

头指针：指向第一个元素所在的结点，起标识一个单链表的作用

尾标志：最后一个元素所在的终端结点的指针域为NULL

其他链表

循环链表

附设头结点，空表和非空表处理一致

没有明显尾端（P != first）

开始结点：first->next

带尾指针时

开始结点：rear->next->next

终端结点：rear

双链表

快速求直接前驱

空间换时间

静态链表

间接寻址：结合数组和指针，将数组中存储数据元素的单元改为存储指向该元素的指针

优点：插入删除时只需要修改指针不移动元素

缺点：没解决连续存储分配的问题，还需要维护一个空闲链，不能随机存取

字符串和多维数组 S=“s1 s2 … sn”

是数据元素为单个字符的线性表

最好情况下时间复杂度O(n+m)

矩阵的压缩存储

行优先：以行序为主序
特殊矩阵：很多值相同的元素且他们的分布有规律，稀疏矩阵不属于特殊矩阵
压缩存储：值相同的元素只分配一个存储空间，零元素不分配存储空间

对称矩阵

关于主对角线对称，只存储下三角部分（和主对角线）
n*n -> n(n+1)/2
存在数组SA[n(n+1)/2]中，下标k=i(i-1)/2+j-1（下标从0开始）

三角矩阵

对称矩阵，对角线上为常数c
n*n -> n(n+1)/2+1
存在数组SA[n(n+1)/2+1]中，下标为k（从0开始）

下三角

当 i >= j 时 k = i(i-1)/2+j-1
当 i < j 时 k = n(n+1)/2

上三角

当 i <= j 时 k = (i-1)(2n-i+2)/2+j-i
当 i > j 时 k = n(n+1)/2

对角矩阵

k = 2 + 3(i-2) + (j-i+2)

稀疏矩阵

非零元素分布没有规律

row，col，item

十字链表
row，col，item，right，down

树和二叉树

树的术语

树：n>=0个结点的有限集合

空树：n=0

非空树：有且仅有一个根节点，其余结点分为m个互不相交的集合，每个集合都是这个根节点的一棵子树

结点：数据元素

度：子树的个数为结点的度，各节点的最大值为该树的度

叶子结点（终端结点）：度为0的结点

分支结点（非终端结点）：度不为0

路径长度：路径上进过的边数，在树中路径是唯一的

结点层数（树的深度、高度）：根结点的层数为1

同层编号：每层从左到右从1开始编号

有序树：从左到右有次序

森林：任何一个数删除根结点就变成森林了，一棵树也可称为森林

遍历

前序：先根后括号

中序：左根右

后序：先左右后根

层序：从第一层，逐层遍历

二叉树的逻辑结构

二叉树的定义

n >= 0

不存在度大于2的结点

是有序的（本书）

斜树：所有结点都只有左子树或右子树，每层只有一个结点

满二叉树：叶子结点只出现在下一层，只有度为0和度为2的结点

完全二叉树：叶子结点只出现在下面俩层，如果有度为1的结点，只能有一个且该结点只有左孩子

满二叉树必是完全二叉树

二叉树的性质

1、第i层上最多有$2^{i-1}$个

2、深度为k的二叉树最多有$2^k-1$个结点，最少有k个结点

3、叶子结点个数为$n_0$，度为2的结点个数为$n_2$，则$n_0=n_2+1$

4、具有n个结点的完全二叉树深度为$\lfloor log_2n \rfloor+1$

5、完全二叉树从1开始按层序编号

如果i>1，结点i的双亲编号为$\lfloor i/2 \rfloor$

如果2i<=n，结点i的左孩子编号为2i

如果2i+1<=n，结点i的右孩子编号为2i+1
